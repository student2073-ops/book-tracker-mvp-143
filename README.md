Simple application to track read books
# Book Tracker MVP

## Проблема
Люди часто забувають, які книги вони вже прочитали, які планують прочитати та на якому етапі читання знаходяться. Звичайні нотатки або списки не завжди зручні для відстеження прочитаних книг.

## Пітч
Book Tracker - це простий застосунок, який допомагає користувачам записувати книги, які вони читають або планують прочитати, переглядати список книг і позначати книги як прочитані. Це дозволяє зручно відстежувати власну історію читання.

## Мета першої ітерації
Створити мінімальну версію застосунку, яка дозволить користувачу:
- додати книгу
- побачити книгу у списку
- позначити книгу як прочитану

Головна мета - перевірити, чи зручно користувачу відстежувати прочитані книги за допомогою такого інструменту.

## MVP (мінімально життєздатний продукт)

Мінімальна версія продукту включає один основний сценарій:

1. Користувач відкриває застосунок
2. Додає назву книги
3. Книга з’являється у списку
4. Користувач може позначити книгу як прочитану

## Карта стейкхолдерів

Основні зацікавлені сторони проєкту:

Користувач - людина, яка хоче відстежувати прочитані книги.  
Розробник - студент, який створює застосунок.  
Викладач - оцінює навчальний проєкт.  
Потенційні користувачі - можуть залишити відгук про зручність застосунку.

## Користувацький сценарій

Початкові умови: користувач відкрив застосунок.

Кроки:

1. Користувач натискає кнопку «Додати книгу».
2. Вводить назву книги.
3. Зберігає запис.
4. Книга з’являється у списку.
5. Після прочитання користувач позначає книгу як прочитану.

Результат: користувач бачить список своїх книг і може відстежувати прочитані.

## Критерії приймання

Мінімальна версія продукту вважається робочою, якщо:

- користувач може додати книгу
- книга з’являється у списку
- книгу можна позначити як прочитану
- новий користувач може зрозуміти інтерфейс без інструкції

## Критерії успіху (Definition of Success)

Проєкт вважається успішним, якщо:

- користувач може додати та позначити щонайменше 3 книги
- основний сценарій виконується без помилок
- користувач розуміє інтерфейс без пояснень
- застосунок працює стабільно під час простого тестування

## План перевірки гіпотези

Гіпотеза: людям зручно використовувати простий інструмент для відстеження прочитаних книг.

Спосіб перевірки:

- користувач додає кілька книг
- позначає одну книгу як прочитану
- після використання залишає короткий відгук

## Наступні кроки

Після перевірки MVP можна додати:

- рейтинг книг
- нотатки до книг
- категорії книг
- пошук у списку
